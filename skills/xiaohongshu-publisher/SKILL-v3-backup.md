# 小红书图文笔记一键发布 v4.0（RedInk风格简化版）

## 核心理念

**简单就是好**：
- ❌ 不用复杂的英文分层prompt
- ❌ 不用颜色代码（#1A1A2E会显示在图上）
- ✅ 用简单中文描述
- ✅ 封面图作为参考图传给内容页，保持风格一致

---

## 三种模式

| 模式 | 触发条件 | 输入 | 输出 |
|------|----------|------|------|
| **资料改写** | 用户提供素材 | 链接/PDF/文字/图片 | 完整笔记 |
| **热点创作** | 用户说"热点"、"爆款" | 品类/方向 | 热点分析 + 笔记 |
| **互动共创** | 用户说"一起写" | 模糊想法 | 对话引导 → 笔记 |

## 图片生成核心方法（RedInk风格）

### 封面图生成

```
请生成一张小红书风格的封面图，尺寸1080x1440像素。

【标题内容】
{标题文字}

【设计要求】
1. 整体风格：小红书爆款封面，清新精致有设计感
2. 背景色：{中文描述颜色}，不要出现颜色代码
3. 标题排版：
   - 标题要大、要醒目，占画面50%以上
   - 字体清晰，一眼能看清
   - 可以把关键词放大突出
4. 装饰元素：简单点缀
5. 可选：角落放Doggie柴犬IP（见下方说明）

【注意事项】
- 标题必须清晰可读
- 不要显示颜色代码或技术参数
```

### 内容图生成（关键：参考图机制）

```
请生成一张小红书风格的内容页图片，尺寸1080x1440像素。

【页面内容】
{正文内容}

【页面信息】
- 页面类型：内容页（第{X}页，共{Y}页）
- 参考风格：请参考封面图的风格，保持统一

【设计要求】
1. 整体风格：与封面保持一致
2. 背景色：{与封面相同}
3. 文字排版：清晰可读，有呼吸感
4. 页码：右下角显示"{X}/{Y}"
5. 可选：角落放Doggie（与封面风格一致）

【注意事项】
- 文字必须清晰可读
- 不要显示颜色代码
- 保持与封面风格统一
```

### ⚠️ 参考图机制（必须执行）

```
Step 1: 生成封面图
→ 保存封面图路径

Step 2: 生成内容页时
→ 在image_generate工具中传入 ref_images=[封面图路径]
→ 在prompt中强调"请参考封面图的风格"
```

## 执行流程

### Phase 1：内容规划

```
1. 分析素材/热点 → 提取核心要点
2. 确定标题（数字要与内容数量匹配！）
3. 规划分页：封面 + 内容页 + 尾页
```

### Phase 2：生成封面图

使用上面的封面图模板生成，保存路径。

### Phase 3：生成内容图

**关键**：每页都传入封面图作为参考图！

```
封面图路径：./xiaohongshu-output/封面.png

内容页1：ref_images=["./xiaohongshu-output/封面.png"]
内容页2：ref_images=["./xiaohongshu-output/封面.png"]
...
```

### Phase 4：文案与话题

```
【正文文案】
{钩子段} + {核心内容} + {行动引导}

【话题标签】（5-8个）
#核心话题 #细分话题 #场景话题

【SEO关键词】
主词：xxx | 长尾：xxx、xxx
```

### Phase 5：发布

调用 myaibo API 发布到草稿箱，返回二维码让用户扫码确认。

## 避坑指南

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 颜色代码显示在图上 | Prompt写了#1A1A2E | 用中文描述颜色 |
| Doggie不一致 | 英文描述太复杂 | 用简单中文描述 |
| 风格不统一 | 每页独立生成 | 封面图传给所有内容页 |
| 标题数字不匹配 | 先定标题再凑内容 | 先统计内容再定标题 |
