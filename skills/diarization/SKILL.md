# Diarization - 结构化档案抽取

## 核心理念

**Diarization是AI系统的核心能力**：
- 从非结构化来源中抽取结构化档案
- 暴露并保留矛盾
- 标出时间上的变化
- 输出Analyst judgment，不是数据库检索

> "SQL查询做不出这个。RAG pipeline做不出这个。模型必须真的读、在脑中同时保留矛盾、注意到什么变了以及何时变的。"

---

## 调用方式

```bash
/diarization
  --target <对象>        # 必填：要分析的专家/主题/人物
  --sources <来源列表>   # 必填：信源文件或URL列表，逗号分隔
  --focus <关注维度>     # 选填：观点|立场|预测|方法论|关系，默认观点
  --output <输出格式>    # 选填：markdown|json|表格，默认markdown
  --timeline <布尔>      # 选填：是否生成时间线，默认true
```

### 参数说明

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| target | ✅ | - | 分析对象（专家名/主题/人物） |
| sources | ✅ | - | 信源列表，支持文件路径或URL |
| focus | ❌ | 观点 | 关注维度：观点、立场、预测、方法论、关系 |
| output | ❌ | markdown | 输出格式：markdown、json、表格 |
| timeline | ❌ | true | 是否生成观点变化时间线 |

---

## 执行流程

### Phase 1：读取信源

```
1. 遍历所有sources
2. 提取每个信源的核心内容
3. 标注来源、时间、可信度
```

### Phase 2：抽取档案

```
对每个信源执行：
1. 识别核心观点/立场/预测
2. 提取支持证据
3. 标注时间戳
4. 记录上下文
```

### Phase 3：对比分析（Judgment核心）

```
跨信源对比：
1. 发现矛盾点 → 标注CONTRADICTION
2. 发现变化 → 标注TIMELINE
3. 发现关联 → 标注CONNECTION
4. 发现缺失 → 标注GAP
```

### Phase 4：结构化输出

```
按照模板输出结构化档案
包含：基本信息、观点矩阵、矛盾分析、时间线
```

## Judgment规则（判断逻辑）

### 如何识别矛盾

1. **时间矛盾**：同一主题在不同时间说不同的话
   - 标注：`CONTRADICTION (时间)`

2. **内容矛盾**：同一时间对不同受众说不同的话
   - 标注：`CONTRADICTION (内容)`

3. **行为矛盾**：说的和做的不一致
   - 标注：`CONTRADICTION (行为)`

### 如何判断重要性

优先级排序：
1. 核心领域观点 > 边缘领域观点
2. 近期观点 > 远期观点
3. 正式场合 > 非正式场合
4. 一贯立场 > 偶尔表态

### 如何处理不确定性

- 缺乏证据时标注 `[待验证]`
- 信息冲突时保留所有版本，不强行统一
- 时间不确定时标注 `[时间?]`

---

## Constraints（边界）

### 禁止行为

- ❌ 编造不存在的观点
- ❌ 强行统一矛盾（保留矛盾才是价值）
- ❌ 省略来源标注
- ❌ 主观判断代替证据

### 失败模式

- 信息过少 → 标注 `[信息不足]`，不强行填充
- 信源冲突 → 保留所有版本，标注可信度
- 无法验证 → 标注 `[待验证]`

## 使用示例

### 示例1：分析专家观点演变

```bash
/diarization
  --target "Ilya Sutskever"
  --sources "./AI专家信源/2026-04-10.md,./AI专家信源/2026-04-11.md"
  --focus 观点
  --timeline true
```

**输出**：
```markdown

## 基本信息
- 对象：Ilya Sutskever
- 身份：OpenAI联合创始人、前首席科学家
- 领域：AI、深度学习、AGI
- 分析日期：2026-04-13
- 信源数量：2

## 核心观点矩阵

| 维度 | 观点 | 证据来源 | 时间 | 置信度 |
|------|------|----------|------|--------|
| Scaling Law | 触顶了 | 多伦多演讲 | 2026-04-06 | 高 |
| AI能力 | 将掌握所有人类任务 | 多伦多演讲 | 2026-04-06 | 高 |
| 人机关系 | 人类不应成为旁观者 | 多伦多演讲 | 2026-04-06 | 高 |

## 矛盾分析

### ⚠️ 内部矛盾
- 之前主张"继续scaling"，现在说"触顶"
- 时间点：2025-2026年间观点转变

```
2025年中 → "Scaling是关键路径"
    ↓

```
## 分析师判断

```
