# 小红书图文笔记一键发布 v5.0（参数化版）

## 核心理念

**技能是方法调用**：
- 同一套流程，不同参数 → 不同能力表现
- 参数化、可复用、与上下文无关

---

## 调用方式

```bash
/xiaohongshu-publisher
  --mode <模式>        # 必填：资料改写|热点创作|互动共创
  --style <风格>       # 选填：RedInk|经典|极简，默认RedInk
  --category <品类>    # 选填：AI科技|生活日常|情感心理|职场事业|健身运动|美妆时尚，默认AI科技
  --doggie <布尔>      # 选填：true|false，默认true
  --background <背景>  # 选填：自定义背景色描述
```

### 参数说明

| 参数 | 必填 | 默认值 | 可选值 |
|------|------|--------|--------|
| mode | ✅ | - | 资料改写、热点创作、互动共创 |
| style | ❌ | RedInk | RedInk、经典、极简 |
| category | ❌ | AI科技 | AI科技、生活日常、情感心理、职场事业、健身运动、美妆时尚 |
| doggie | ❌ | true | true、false |
| background | ❌ | 自动匹配品类 | 自定义中文描述 |

---

## 三种模式

### mode=资料改写
用户提供素材（链接/PDF/文字/图片）→ 提取改写 → 完整笔记

### mode=热点创作
用户说"热点"、"爆款"、"找选题" → 自动搜索 → 热点分析 + 笔记

### mode=互动共创
用户说"一起写"、"帮我想" → 对话引导 → 笔记

---

## 三种风格

### style=RedInk（推荐）
- 简单中文prompt
- 不用颜色代码
- 参考图机制
- 适合大多数场景

### style=经典
- 三段式结构
- 更多装饰元素
- 适合需要丰富视觉的场景

### style=极简
- 最小化设计
- 文字为主
- 适合专业内容

## Doggie IP 参数

### doggie=true（默认）
在图片角落添加Doggie柴犬IP

**简化描述（中文）**：
```
一只可爱的Q版柴犬卡通形象：
- 橘黄色毛发，白色脸蛋花纹
- 圆圆的脸，三角形的立耳朵
- 黑豆眼，小黑鼻，粉色腮红
- 穿着{服装}，表情{表情}
- 大小：占图片高度15-20%
- 位置：右下角或左下角
```

### doggie=false
不添加Doggie，纯文字设计

## 图片生成模板（根据参数动态生成）

### 封面图模板

```
请生成一张小红书风格的封面图，尺寸1080x1440像素。

【标题内容】
{标题文字}

【设计要求】
1. 整体风格：{style对应风格}
2. 背景色：{background 或 category对应背景色}
3. 标题排版：
   - 标题要大、要醒目，占画面50%以上
   - 字体清晰，一眼能看清
   - 可以把关键词放大突出
4. 装饰元素：简单点缀
{doggie=true时添加}
5. 角落放Doggie柴犬IP：{category对应服装和表情}

【注意事项】
- 标题必须清晰可读
- 不要显示颜色代码或技术参数
```

### 内容图模板

```
请生成一张小红书风格的内容页图片，尺寸1080x1440像素。

【页面内容】
{正文内容}

【页面信息】
- 页面类型：内容页（第{X}页，共{Y}页）
- 参考风格：请参考封面图的风格，保持统一

【设计要求】
1. 整体风格：与封面保持一致
2. 背景色：{与封面相同}
3. 文字排版：清晰可读，有呼吸感
4. 页码：右下角显示"{X}/{Y}"
{doggie=true时添加}
5. 角落放Doggie：与封面风格一致，{页面类型对应动作}

【注意事项】
- 文字必须清晰可读
- 不要显示颜色代码
- 保持与封面风格统一
```

## 参考图机制（必须执行）

```
Step 1: 生成封面图
→ 保存封面图路径

Step 2: 生成内容页时
→ 在image_generate工具中传入 ref_images=[封面图路径]
→ 在prompt中强调"请参考封面图的风格"
```

## 执行流程

### Phase 1：解析参数

```
输入：用户消息
解析：
- mode：识别模式
- style：默认RedInk
- category：默认AI科技
- doggie：默认true
- background：从category映射或用户指定
```

### Phase 2：内容规划

```
1. 分析素材/热点 → 提取核心要点
2. 确定标题（数字要与内容数量匹配！）
3. 规划分页：封面 + 内容页 + 尾页
```

### Phase 3：生成图片

```
根据参数生成：
- 应用 style 对应的prompt模板
- 应用 category 对应的背景色和Doggie配置
- 应用 doggie 开关

封面图 → 保存路径
内容页1-N → 传入封面图作为参考图
```

### Phase 4：文案与话题

```
【正文文案】
{钩子段} + {核心内容} + {行动引导}

【话题标签】（5-8个）
#核心话题 #细分话题 #场景话题

【SEO关键词】
主词：xxx | 长尾：xxx、xxx
```

### Phase 5：发布

调用 myaibo API 发布到草稿箱，返回二维码让用户扫码确认。

## Judgment & Constraints（判断与边界）

### 何时用什么风格
- **RedInk**：大多数场景，尤其需要简洁清晰时
- **经典**：需要丰富视觉、装饰元素时
- **极简**：专业内容、文字为主时

### 何时用Doggie
- **true**：个人IP内容、轻松话题、需要差异化时
- **false**：正式内容、品牌合作、需要严肃感时

### 禁止行为
- ❌ 先定标题再凑内容（先统计内容再定标题）
- ❌ 使用颜色代码（#1A1A2E会显示在图上）
- ❌ 内容页不传参考图（风格不统一）
- ❌ 标题数字与内容数量不匹配

### 失败模式
- 图片生成超时 → 简化prompt，分批生成
- Doggie不一致 → 使用参考图机制
- 风格不统一 → 确保每页传入封面图
